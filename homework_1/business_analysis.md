### 🔹 Бизнес-цель проекта
**Оптимизировать процесс затоваривания складов и торговых точек сети за счёт снижения двух типов потерь**:
1. **Потери от порчи** (over-forecasting для скоропортящихся товаров: овощи, молочка, мясо)
2. **Упущенная выручка** (under-forecasting для товаров с долгим сроком хранения и высокой маржой)

**Целевой эффект (гипотезы для валидации)**:
- Снижение симулированных бизнес-потерь на **10–15%** относительно базового прогноза (наивный метод / Prophet)
- Сохранение уровня сервиса **≥95%** (доля заказов, покрытых прогнозом)

> *Примечание: Конкретные цифры будут уточнены после EDA и анализа распределения ошибок на валидационной выборке. В реальном проекте они согласовываются с заказчиком на основе исторических потерь.*

### 🔹 Существующие решения и их ограничения
| Решение | Как работает | Почему недостаточно |
|---------|-------------|---------------------|
| **Ручное планирование** | Менеджер опирается на опыт + скользящее среднее за прошлые недели | Не масштабируется, субъективно, не учитывает асимметрию рисков |
| **Простые статистические модели** (ARIMA, Exponential Smoothing) | Точечный прогноз на основе истории | Не дают интервалов неопределённости, не адаптируются под тип товара |
| **ML-модели без учёта риска** (LightGBM, XGBoost на MSE) | Минимизируют среднеквадратичную ошибку | Штрафуют ошибку «вверх» и «вниз» одинаково, хотя бизнес-цена разная |

**Наше отличие**: явное моделирование **асимметричной функции потерь** через квантильный прогноз и иерархическую агрегацию.

### 🔹 Риски и план смягчения
| Риск | Вероятность | Влияние | План смягчения |
|------|-------------|---------|----------------|
| **Не уложиться в сроки** | Средняя | Высокое | Итеративный подход: сначала простой baseline, затем усложнение |
| **Мало данных по новым товарам/точкам** | Высокая | Среднее | Иерархический пуллинг: прогноз на уровне `dept`/`category`, если `item` неустойчив |
| **Сезонность и внешние шоки** | Средняя | Высокое | Использовать календарь M5 (праздники, SNAP, события) как ковариаты |
| **Переобучение на шум** | Низкая | Среднее | Валидация по времени (time-series split), ранняя остановка, регуляризация |
| **Сложность интерпретации для бизнеса** | Средняя | Среднее | Визуализация: «что если»-сценарии, объяснение персентилей через примеры товаров |

## 1.2 Решаемые задачи с точки зрения аналитики (Data Mining goals) 

### 🔹 Технические цели
| Задача | Описание |
|--------|----------|
| **Прогноз спроса** | Квантильный прогноз на 28 дней вперёд для каждой комбинации `store_id × item_id` |
| **Учёт асимметрии потерь** | Для скоропортящихся товаров — прогноз на уровне 40–45-го перцентиля; для долгохрана — 85–90-го |
| **Агрегация на уровень склада** | Суммирование квантилей по точкам с учётом типа товара и маржинальности |
| **Обработка «холодных» точек** | Если данных < 30 дней — использовать медиану по кластеру похожих магазинов (`state_id + dept_id`) |

### 🔹 Метрики оценки
| Тип | Метрика | Обоснование                                                                                              | Статус |
|-----|---------|----------------------------------------------------------------------------------------------------------|--------|
| **Основная (соревновательная)** | `WRMSSE` (Weighted Root Mean Squared Scaled Error) | Официальная метрика M5, учитывает иерархию и масштаб товаров                                             | Обязательно |
| **Для обучения модели** | `Quantile Loss (Pinball)` | Позволяет получать квантильные прогнозы; `α` выбирается под тип товара (скоропорт/долгохран)             | Обязательно |
| **Бизнес-оценка** | `Simulated Cost Loss` | `Loss = C_spoilage·max(0,pred−actual) + C_margin·max(0,actual−pred)`. Показывает финансовый эффект       | Обязательно |
| **Калибровка неопределённости** | `Interval Coverage (80%)` | Проверка, что 80%-ный интервал действительно покрывает ~80% фактов. Показывает надёжность оценки рисков. | Опционально |

### 🔹 Критерии успешности
- `WRMSSE` на тесте ≤ 0.95 относительно официального baseline M5
- Снижение симулированных бизнес-потерь ≥10% против наивного прогноза
- Воспроизводимость: код, данные и артефакты версионированы через DVC, эксперименты отслежены, пайплайн документирован
- Покрытие 80% фактических продаж внутри предсказанного 80%-ного интервала (опционально)
