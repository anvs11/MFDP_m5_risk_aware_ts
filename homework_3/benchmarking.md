## 1. Выбор конкурентов

### Критерии выбора
Одной из поставленных задач был поиск решений, которые закрывают потребность в **прогнозировании спроса и оптимизации запасов** для ритейла, с фокусом на скоропортящиеся товары.
Помимо указанных в анализе есть еще сильные прямые конкуренты, но информация по ним (цены, отзывы, метрики) часто закрыта или требует демо-доступа. Для учебного проекта берём решения с открытыми данными.

### Выбранные конкуренты

| Конкурент                            | Тип       | Обоснование выбора                                                                                                                               |
|--------------------------------------|-----------|--------------------------------------------------------------------------------------------------------------------------------------------------|
| **1C:Универсальное прогнозирование** | Прямой    | Массовое решение в РФ для малого/среднего ритейла. Закрывает ту же потребность (прогноз спроса) тем же способом (статистика + ML) для той же ЦА. |
| **KORUS Forecast**                   | Прямой    | Облачный сервис для FMCG-ритейла. Прямой аналог по функционалу и ЦА, но с акцентом на «высокую точность» и работу с сырыми данными.              |
| **Excel + ручное планирование**      | Косвенный | То, с чем реально конкурирует прототип на старте. Закрывает ту же потребность (планирование заказа), но иным способом (ручной труд).             |
| **RELEX Solutions**                  | Прямой    | Глобальный лидер в retail forecasting. Показывает отраслевой стандарт точности, управления риском и работы с иерархией данных.                   |

### Подход лидеров рынка (Walmart)

Walmart не использует сторонние forecasting-инструменты для ядра системы — они разрабатывают решения internally (например, участвовали в создании датасета M5, публикуют исследования по ML в supply chain).

Однако их подход полезен как **отраслевой бенчмарк** для моего проекта:

| Аспект                     | Подход Walmart (на основе публикаций)                                           |
|----------------------------|---------------------------------------------------------------------------------|
| **Гранулярность прогноза** | `item × store × day` с иерархической агрегацией                                 |
| **Ковариаты**              | Учёт промо, календарных событий, погоды как внешних факторов                    |
| **Метрика**                | `WRMSSE` — официальная метрика M5-соревнования                                  |
| **Управление риском**      | Фокус на uncertainty: квантильные прогнозы для управления риском порчи/дефицита |

**Вывод для проекта**: Даже лидер рынка (Walmart) инвестирует в **квантильное прогнозирование** и **явное управление риском** — это подтверждает актуальность выдвинутой гипотезы: асимметричная функция потерь через выбор перцентиля может дать измеримый бизнес-эффект для скоропортящихся товаров.

## 3. Анализ и сравнение

### Ответы на ключевые вопросы

**1. Какие функции и особенности являются обязательными для вашего продукта, исходя из анализа конкурентов?**
- Прогноз на уровне `товар×магазин×день`;
- Учёт сезонности, промо, календарных событий как ковариат;
- Базовая проверка данных на аномалии (выбросы, пропуски, отрицательные значения);
- Экспорт результатов в базовом формате (CSV/Excel) для передачи в рабочие процессы (заказ поставщику, 1C, почта);
- Визуализация прогноза (график + ключевые метрики);
- Базовая оценка точности (MAPE/WRMSSE) для доверия к результату

**2. Какие функции вы можете предложить, чтобы отличаться от конкурентов?**
- Асимметричная функция потерь: управление риском через выбор перцентиля под тип товара (конкуренты используют MSE/MAPE, штрафующие ошибки «вверх» и «вниз» одинаково);
- Интерпретируемость: не просто число, а «Заказ: X шт. (риск порчи: низкий/средний/высокий)»;
- Фокус на скоропорт: специализация на категории с высокой ценой ошибки, а не универсальность «для всех»;
- Простота внедрения: запуск за 1 день, без обучения и доработок (vs 1C/RELEX)

**3. Какие недостатки у конкурентов вы можете устранить в своем продукте?**
- 1C: сложный интерфейс, долгое внедрение → наш простой Streamlit-интерфейс + быстрый старт;
- Excel: ошибки в формулах, нет контроля версий → у нас DVC + MLflow, воспроизводимость по хешу;
- KORUS/RELEX: высокая цена, зависимость от интернета → наш прототип работает локально/в Colab, гипотетическая цена ≤$200/мес;
- Все: точечный прогноз без управления риском → у нас квантильный прогноз с явным выбором α под срок годности

**4. Какие стратегии монетизации являются наиболее эффективными и почему?**
- Для прототипа: usage-based ($0.01/прогноз) — низкий порог входа, легко тестировать гипотезу;
- Для масштабирования: SaaS-подписка ($50–200/мес за точку) — предсказуемый доход, соответствует модели KORUS и ниже enterprise-решений;
- Обоснование: подписка доминирует в нише (KORUS, RELEX), но usage-based снижает риск для клиента на старте. Гибридная модель (пилот → подписка) — оптимальный путь

**5. Какой пользовательский опыт можно улучшить в вашем продукте по сравнению с конкурентами?**
- Скорость: от загрузки данных до прогноза — ≤5 мин (в 1C: часы/дни на настройку);
- Понятность: визуализация риска и бизнес-рекомендация, а не только график (в Excel: только цифры);
- Доверие: показ интервалов и симуляция потерь в рублях (в конкурентах: только техническая точность);
- Гибкость: переключатель «скоропорт/долгохран» меняет α на лету (в конкурентах: фиксированная логика) 

**6. Какими ключевыми характеристиками должен обладать ваш продукт, чтобы быть конкурентоспособным?**
- Точность: WRMSSE ≤0.95, MAPE ≥75% (отраслевой стандарт);
- Снижение потерь: ≥10% (наша гипотеза, бенчмарк кейса Coop);
- Время инференса: ≤2 сек/прогноз (быстрее 1C);
- Fill Rate: ≥95% (отраслевой SLA);
- Цена: ≤$200/мес за точку (дешевле enterprise);
- Воспроизводимость: 100% артефактов в DVC/MLflow

## 4. Определение стандартов

**1. Минимальные требования к функциональности:**
- Прогнозирование спроса на уровне товар×магазин×день;
- Учёт сезонности, промо, календарных событий как ковариат;
- Базовая валидация данных: проверка на аномалии, пропуски, отрицательные значения;
- Экспорт рекомендаций в формате CSV/Excel (совместимость с 1C, почтой, внешними системами);
- Визуализация прогноза: график + интервал неопределённости + ключевые метрики;
- Гибкий горизонт прогноза: 7 дней для скоропорта, 28 дней для долгохрана;
- Базовая оценка точности: расчёт MAPE/WRMSSE на валидационной выборке

**2. Ключевые конкурентные преимущества:**
- Асимметричная функция потерь: управление риском через выбор перцентиля под тип товара (конкуренты используют MSE/MAPE, штрафующие ошибки «вверх» и «вниз» одинаково);
- Интерпретируемость: «Заказ: X шт. (риск порчи: низкий/средний/высокий)» вместо сухого числа — менеджер видит бизнес-смысл прогноза
- Фокус на скоропортящихся товарах: специализация на категории с высокой ценой ошибки, а не универсальность «для всех»
- Простота внедрения: запуск за 1 день, без обучения и доработок (vs 1C/RELEX)
- Воспроизводимость: все артефакты версионированы (DVC + MLflow) — любой результат можно восстановить по хешу

**3. Оптимальная модель ценообразования и монетизации:**
